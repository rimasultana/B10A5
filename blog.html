What is the document Object model (dom)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a web page in a tree-like format, where each part of the document (like elements, attributes, and text content) is a node that can be manipulated by programming languages such as JavaScript.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the context of the DOM where a single event listener is added to a parent element instead of adding multiple listeners to individual child elements. The event listener takes advantage of event bubbling (the process by which events propagate up from the target element to the parent elements) to handle events on child elements dynamically.

Event Bubbling: When an event occurs on an element, it first runs any handlers on that element, then "bubbles" up to its parent, and so on up to the root of the DOM. By listening on the parent, you can capture events from any of its child elements. Using target Property: The event object has a target property that tells you which child element triggered the event. This allows you to determine which child element was clicked or interacted with, even though the listener is on the parent.
